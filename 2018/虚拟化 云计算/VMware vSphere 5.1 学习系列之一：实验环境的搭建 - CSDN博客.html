VMware vSphere 5.1 学习系列之一：实验环境的搭建

转载 2013年10月28日 09:45:01
  • 编辑
  • 删除

一、 物理主机配置

我的物理主机硬件配置如下：

主板

Intel DQ67SW

支持VT-D功能，在VMware Workstation下无法使用它

CPU

Intel i7 2600

支持VT-x/EPT功能，可实现64位系统的嵌套

内存

4*4 GB

满足基本实验的内存需求，32GB将获得更好性能

硬盘

3*320 GB RIAD 0

实验用，需要硬盘速度，不考虑数据安全

二、 实验环境规划

实验环境规划详情

实验环境规划示意图：

实验环境规划说明：

我们用VMware Workstation搭建实验环境，一个基本环境共需6台虚拟机，分别是：3台ESXi、1台vCenter、1台AD&DNS、1台Openfiler。根据实验内容的不同，还会增加虚拟机的数量。

ESXi和vCenter虚拟机，使用VMware vSphere 5.1版本软件。

AD&DNS和vCenter虚拟机，使用Windows Server 2008 R2 SP1系统，域名为jack.com。

Openfiler虚拟机，使用Openfiler NAS/SAN 2.99.1版本软件。

各虚拟机间的网络连接，使用Bridged挢接模式。

大家可以根据实验环境规划示意图，配置好各虚拟机的硬件、网络、系统等环境。

大家需要注意的是，使用VMware Workstation构建的虚拟实验环境，虽然可以完成绝大多数的vSphere功能实验，但对于一些对硬件有特定要求的功能，就无能为力了。

设置VMware Workstation内存选项

从实验环境规划示意图中，不难发现一个问题，所有虚拟机总共需要19GB内存，而物理主机只有16GB内存（实际可用物理内存的会更小），显然物理内存已经不够用了。

虽然分配了19GB，但在虚拟机运行时，不会完全用到19GB，可以调整一下VMware Workstation内存工作机制，以满足实验环境的运行需求。

运行VMware Workstation程序，在主窗口的Home标签页中，点击Workstation Preferences。

在Preferences窗口中，选择Memory栏。

在Additional memory区域，选择Allow most virtual machine memory to be swapped，点击OK，保存配置。

三、 创建虚拟机

创建VMware Workstation虚拟机

根据实验环境规划，我们来创建一个虚拟机esx1.jack.com。

在VMware Workstation窗口的Home标签页中，点击Create a New Virtual Machine。

打开New Virtual Machine Wizard窗口。

在Welcome to the New Virtual Machine Wizard页，将What type of configuration do you want? 选择为Custom (advanced)，点击Next。

在Choose the Virtual Machine Hardware Compatibility页，将Hardware compatibility选择为Workstation 9.0，点击Next。

在Guest Operating System Installation页，将Install from选择为I will install the operating system later，点击Next。

在Select a Guest operating system页，将Guest operating system选择为VMware ESX，将Version选择为VMware ESXi 5，点击Next。

在Name the Virtual machine页，在Virtual machine name处输入虚拟机名称，在Location处浏览或输入虚拟机文件保存的位置，点击Next。

在Processors Configuration页，将Number of Processors和Number of cores per processor选择为2，点击Next。

在Memory for the Virtual Machine页，将内存大小滑块调整至4 GB处，点击Next。

在Network Type页，将Network connection选择为Use bridged networking，点击Next。

在Select I/O Controller Types页，将SCSI Controller选择为LSI Logic，点击Next。

在Select a Disk处，将Disk选择为Create a new virtual disk，点击Next。

在Select a Disk Type页，将Virtual disk type选择为SCSI(Recommended)，点击Next。

在Specify Disk Capacity页，在Maximum disk size (GB)中输入40.0，并选择Store virtual disk a single file，点击Next。

在Specify Disk file页，在Disk file中使用默认的虚拟机磁盘文件名称，点击Next。

在Ready to Create Virtual Machine页，查看虚拟机配置信息，点击Finish。

修改VMware Workstation虚拟机配置

在VMware Workstation窗口中，可以看到刚刚创建成功的虚拟机esx1.jack.com。

根据实验环境规划，我们现在还需要对esx1.jack.com虚拟机的硬件进行添加和删除，点击Edit virtual machine settings。

打开Virtual Machine Settings窗口，在Hardware标签页，选择Processors栏，在Virtualization engine处勾选择Virtualize Intel VT-x/EPT or AMD-V/RVI和Virtualize CPU performance counters。

选择CD/DVD (IDE) 栏，在Connection处，选择Use ISO image file，并输入或选择ESXi 5安装光盘映像ISO文件。

选择Floppy栏，点击Remove，删除此虚拟硬件。

点击Add，添加新虚拟硬件。

打开Add Hardware Wizard窗口。

在Hardware Type页，将Hardware选择为Network Adapter，点击Next。

在Network Adapter Type页，将Network connection选择为Bridged: Connected directly to the physical network，点击Finish。

在Hardware标签页中，可以看到新添加的网络适配器Network Adapter 2。

根据实验环境规划，继续添加其它的Network Adapter网络适配器。

添加操作完成后，点击OK。

在VMware Workstation窗口中，可以看到虚拟机esx1.jack.com硬件修改后的结果。

根据实验环境规划，按照虚拟机esx1.jack.com的创建过程，完成虚拟机esx2.jack.com、esx3.jack.com、dc.jack.com、vc.jack.com、open.jack.com的创建工作。
